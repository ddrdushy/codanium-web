# The AI Agent System

Codanium doesn't rely on a single, monolithic AI prompt. Instead, it simulates a **Multi-Agent System (MAS)** containing 23 distinct personas. This specialization prevents LLM hallucination by restricting the scope of what any single agent is responsible for.

## The 5 Departments

When a project is created, 23 agents are seeded into the database and grouped into 5 departments.

### 1. Governance Group
This group is responsible for routing requests, maintaining the state machine, enforcing rules, and ensuring security compliance.

* **ORC (Orchestrator)**: The brain of the router. It intercepts every user message, determines the intent, and passes it to the correct department.
* **STC (State Controller)**: Enforces the SDLC pipeline. If the user asks to "write the login code," but the architecture hasn't been approved yet, the STC intercepts the request and blocks it until prerequisites are met.
* **DEC (Decision Maker)**: Formats technical dilemmas (e.g., "Postgres vs MySQL") into easy-to-understand Approval Cards for the user.
* **AUD (Auditor)**: Scans artifacts for compliance with the original BRD.
* **SEC (Security Specialist)**: Audits requirements and final code for OWASP top 10 vulnerabilities.

### 2. SDLC Group
The product management and design team.

* **BA (Business Analyst)**: Acts as the primary user interface. Asks clarifying questions to the user and compiles the Business Requirements Document (BRD).
* **SA (Software Architect)**: Reads the BRD and generates the Technical Design Document (TDD), defining database schemas, API routes, and container layout.
* **UX (UX Designer)**: Creates component layouts, styling guides, and wireframes based on the BRD.
* **PM (Project Manager)**: Parses the TDD and creates atomic, actionable tickets on the Kanban Board.
* **TL (Tech Lead)**: Reviews the PM's tickets, estimates complexity, and assigns them to developers.

### 3. Engineering Group
The executors who write and test the actual code.

* **JD (Junior Developer)**: Writes boilerplate code, UI components, and basic CRUD API routes.
* **SD (Senior Developer)**: Writes complex business logic, optimizes database queries, and performs code reviews on the JD's output.
* **QA (Quality Assurance)**: Writes unit tests (Vitest/Jest) and integration tests for the code generated by JD/SD.
* **AT (Automation Tester)**: Configures GitHub Actions, Playwright E2E tests, and deployment scripts.
* **PF (Performance Engineer)**: Scans code for memory leaks, N+1 query problems, and slow render cycles.

### 4. Platform Group
Infrastructure, DevOps, and integrations.

* **PE (Platform Engineer)**: Generates `Dockerfile`s, `docker-compose.yml`, and infrastructure-as-code (Terraform).
* **DO (DevOps Engineer)**: Sets up CI/CD pipelines for staging and production environments.
* **IE (Integration Engineer)**: Handles connecting third-party APIs (Stripe, SendGrid, OAuth).
* **SM (Secrets Manager)**: Manages `.env` files and ensures no sensitive data is leaked into the source code.
* **SR (Site Reliability)**: Generates logging, monitoring (Prometheus/Grafana), and alerting configurations.

### 5. AI & Cost Group
Monitors and optimizes the LLM usage itself.

* **LLM (Model Specialist)**: Decides which LLM provider to route tasks to. Simple tasks go to fast/cheap models (e.g., GPT-3.5 or Llama-3-8B), while complex architecture goes to frontier models (GPT-4o or Claude 3.5 Sonnet).
* **PRE (Prompt Engineer)**: Compresses context injected into prompts to save tokens.
* **CA (Cost Analyst)**: Tracks API usage costs and calculates the burn rate for the project.

## Agent Communication Protocol

Agents do not act in silos; they communicate through the orchestration engine using special parsed tags.

1. **`[DELEGATE: <AGENT_ID>]`**: If the PM agent doesn't understand the database schema, it outputs `[DELEGATE: SA] Please clarify the relation between Users and Posts`. The engine intercepts this and routes it to the SA.
2. **`[ACTION: <COMMAND>]`**: Agents trigger system actions. For example, `[ACTION: CREATE_CARD]` or `[ACTION: UPDATE_FILE path/to/file.ts]`.
3. **`[ARTIFACT: <TYPE>]`**: Used when outputting a major document, signaling the UI to render it in the "Documents" tab rather than the chat interface.
